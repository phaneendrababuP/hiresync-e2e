# hiresync-e2e

Playwright + TypeScript test suite for hiresync. Covers the candidate hiring pipeline,
NLP scoring features, and the REST API layer.

Started this as a set of standalone spec files and gradually pulled shared logic into
page objects and helpers as the test count grew. Currently sitting around 35 test cases
across smoke, regression, and API suites.

---

## What's covered

- Login flows and session handling
- Candidate list — search and filter by status
- Candidate profile — status changes, NLP parse results, Big5 chart rendering
- Assessment dispatch flow (modal open/send/cancel)
- Competency scoring output validation
- API: auth, candidate CRUD, NLP endpoint

---

## Setup

Requires Node 20 and npm.

```bash
npm install
npx playwright install
```

Create the env folder and copy the template:

```bash
mkdir -p env
cp .env.example env/.env.staging
# fill in the values in env/.env.staging
```

---

## Running tests

```bash
# smoke check — run this after any deployment
npm run test:smoke

# full regression
npm run test:regression

# API tests only (no browser)
npm run test:api

# headed mode for local debugging
npm run test:headed

# Playwright UI mode — step through tests visually
npm run test:ui-mode

# debug mode — opens inspector on first test
npm run test:debug
```

To run against dev instead of staging:

```bash
ENV=dev npm run test:smoke
```

---

## Test tags

Tests are tagged so you can run subsets:

- `@smoke` — 5 critical path tests, fast, run on every deployment
- `@regression` — full suite, run on main merges
- `@api` — API-only tests, no browser needed
- `@auth` — login and session tests
- `@candidates` — candidate pipeline tests
- `@ai` — NLP and scoring tests

## Folder structure

```
src/
  pages/        page objects (one file per page)
  fixtures/     test setup — auth state, page injection
  api/          API classes for auth and candidates
  factories/    test data — candidate and user builders
  helpers/      env config, logger, random data
  constants/    routes, timeouts, tags, messages

tests/
  smoke/        critical path tests
  regression/   full test suites by feature
  api/          API-level tests
```

---

## Auth setup

Tests that need a logged-in session use `storageState` files under `.auth/`.
These are generated by `src/fixtures/auth.setup.ts` before the suite runs,
and gitignored so they aren't committed.

The setup skips re-login if the file exists and is less than 55 minutes old,
which speeds up local re-runs.

---

## Environment variables

All config lives in `env/.env.<environment>`. The `.env.example` file has all the keys.
In CI, values come from GitHub Actions Secrets — don't put actual credentials in the file.

---

## Reports

```bash
npm run test:report
```

Opens the Allure report locally. CI publishes results as artifacts after each run.

---

## Notes

NLP confidence scores and competency scores from hiresync's AI features are non-deterministic.
Tests in `tests/regression/assessments/` check ranges and thresholds rather than exact values
for this reason.
